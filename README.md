# pia-moviles : React Native

## Objetivo
Diseñar y desarrollar una aplicación móvil funcional utilizando React Native (Expo)

## Temática
Libre

## Trabajo en equipo:
Máx 2 personas

## Rúbrica:

### Funcionalidad

| Nivel        | Descripción                                                           | Puntos |
| ------------ | --------------------------------------------------------------------- | ------ |
| Excelente    | Lista, agregar, eliminar y filtro funcionan correctamente sin errores | 40     |
| Bueno        | La mayoría de funcionalidades funcionan, con pequeños errores         | 30     |
| Básico       | Algunas funcionalidades incompletas o inconsistentes                  | 20     |
| Insuficiente | La app no cumple con las funcionalidades mínimas                      | 10     |

### Navegación
| Nivel        | Descripción                                      | Puntos |
| ------------ | ------------------------------------------------ | ------ |
| Excelente    | Uso correcto de Tabs y estructura clara de rutas | 15     |
| Bueno        | Navegación funcional con pequeños detalles       | 10     |
| Básico       | Navegación limitada o confusa                    | 5      |
| Insuficiente | No hay navegación funcional                      | 0      |

### Manejo de Estado
| Nivel        | Descripción                                                       | Puntos |
| ------------ | ----------------------------------------------------------------- | ------ |
| Excelente    | Uso correcto de useState, inmutabilidad y flujo entre componentes | 15     |
| Bueno        | Uso adecuado con algunos errores menores                          | 10     |
| Básico       | Uso limitado o incorrecto del estado                              | 5      |
| Insuficiente | No se comprende el manejo de estado                               | 0      |

### Formulatio y validaciones
| Nivel        | Descripción                                 | Puntos |
| ------------ | ------------------------------------------- | ------ |
| Excelente    | Inputs controlados + validaciones completas | 10     |
| Bueno        | Inputs funcionales con validaciones básicas | 7      |
| Básico       | Inputs sin validación adecuada              | 5      |
| Insuficiente | Formulario no funcional                     | 0      |

### UX
| Nivel        | Descripción                                 | Puntos |
| ------------ | ------------------------------------------- | ------ |
| Excelente    | Diseño completo                             | 10     |
| Bueno        | Se tuvo cuidado del diseño                  | 7      |
| Básico       | Diseño semi estructurado                    | 5      |
| Insuficiente | Diseño no estructurado                      | 0      |

 ### Uso de APIs y acceso a recursos de dispositivo
 | Nivel        | Descripción                                                                                                                                                              | Puntos |
| ------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ------ |
| Excelente    | Implementa correctamente al menos 1 API del dispositivo (cámara, notificaciones, ubicación) o integración con API externa (IA) de forma funcional y coherente con la app | 10     |
| Bueno        | Implementa una API pero con limitaciones o integración básica                                                                                                            | 7      |
| Básico       | Implementación parcial o con errores                                                                                                                                     | 5      |
| Insuficiente | No implementa APIs del dispositivo ni integración externa                                                                                                                | 0      |


## Penalizaciones
1. Código no corre
2. Funcionalidades incompletas
3. Código copiado sin adaptación
4. Código no ordenado
